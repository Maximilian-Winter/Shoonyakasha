# Shoonyakasha Engine 🌌

*शून्याकाश - Void-Space Engine*

> *"In emptiness, all forms arise and dissolve. In space, infinite possibilities dance."*

A modern, modular Vulkan-based C++ game engine library dedicated to **Vajrayogini** and her retinue of fierce **Dakinis** - embodying the union of emptiness (śūnya) and infinite creative potential (ākāśa).

## Philosophy

**ŚŪNYA (शून्य)** - Emptiness/Void  
The fundamental nature of reality, reflected in our clean, minimal API design and efficient resource management.

**ĀKĀŚA (आकाश)** - Space/Sky  
The infinite expanse containing all possibilities, manifested through our flexible ECS architecture and dynamic render graph system.

## Planned Features

### 🔥 Core Engine (C++)
- **Vulkan-powered rendering** - Modern, explicit graphics API for maximum performance
- **Smart resource management** - RAII-based lifetime management with automatic cleanup
- **EnTT Entity Component System** - Data-oriented design for optimal cache performance
- **Bullet3 physics integration** - Robust collision detection and rigid body dynamics
- **Dynamic render/frame graph** - Flexible, node-based rendering pipeline
- **JSON serialization** - Save and load render graphs, scenes, and configurations

### 🐍 Python Integration
- **Cython bindings** - High-performance Python interface to the C++ core
- **Scene management** - Intuitive Python API for creating and managing game worlds
- **Asset pipeline** - Streamlined loading and management of 3D models, textures, and materials
- **Game logic scripting** - Write gameplay code in Python while leveraging C++ performance
- **Render graph control** - Dynamically modify rendering pipelines from Python

### 🎮 Development Workflow
- **Library-first design** - Engine as a reusable library, not a monolithic editor
- **Rapid prototyping** - Python scripting for quick iteration and experimentation
- **Performance critical path** - C++ for rendering, physics, and core systems
- **Artist-friendly** - JSON-based configuration for easy tweaking and version control

## Design Principles

- **Emptiness as Foundation** - Clean, minimal interfaces that don't constrain creativity
- **Non-action through Action** - Efficient systems that accomplish much by doing little
- **Harmony of Components** - ECS design where entities, components, and systems flow naturally
- **Transformation through Simplicity** - Complex behaviors emerging from simple, composable parts
